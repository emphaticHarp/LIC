IMPLEMENTATION DETAILS
═══════════════════════════════════════════════════════════════════════════════

HELP PAGE PAGINATION IMPLEMENTATION:

State Management:
- faqCurrentPage: Tracks current FAQ page
- articlesCurrentPage: Tracks current articles page
- ticketsCurrentPage: Tracks current tickets page
- ticketSearchQuery: Search query for tickets
- ticketStatusFilter: Status filter for tickets
- itemsPerPage: Set to 5 items per page

Pagination Logic:
- Calculates total pages: Math.ceil(items.length / itemsPerPage)
- Calculates start index: (currentPage - 1) * itemsPerPage
- Calculates end index: startIndex + itemsPerPage
- Slices data: items.slice(startIndex, endIndex)

UI Components:
- Previous/Next buttons with disabled states
- Page number buttons (1, 2, 3, etc.)
- Results counter: "Showing X to Y of Z items"
- Search and filter inputs
- Maintains state across filter changes

VALIDATION UTILITY USAGE:

Example in new-policy page:
```typescript
import { validateEmail, validatePhone, validateAadhaar } from '@/lib/validation';

const emailResult = validateEmail(formData.applicantEmail);
if (!emailResult.isValid) {
  errors.push(emailResult.error);
}

const phoneResult = validatePhone(formData.applicantPhone);
if (!phoneResult.isValid) {
  errors.push(phoneResult.error);
}
```
